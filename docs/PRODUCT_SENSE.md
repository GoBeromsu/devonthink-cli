# PRODUCT SENSE

The product is useful when it helps users control native DEVONthink automation from the shell without inventing a second product model.

Good product decisions:

- make native capabilities easier to access
- improve discoverability of native properties and commands
- keep output stable and machine-friendly

Bad product decisions:

- inventing helper workflows as first-class runtime features
- hiding native DEVONthink semantics behind custom abstractions
